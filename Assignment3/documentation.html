Retrieve Photos API Documentation

Endpoint

GET /photos

Response

On success, the API will return a 200 status code and an array of photo objects with the following properties:

  • id (number, required) - unique identifier of the photo
  • author (string, required) - author of the photo
  • alt (string, required) - alternative text of the photo
  • tags (string, optional) - tags associated with the photo
  • image (string, required) - URL of the photo
  • description (string, optional) - description of the photo

On error, the API will return a console error with the error message.

Example

Below is an example of a successful retrieval of all photos: 

        GET /photos
        Response:
        200
        [
            {
                "id": 1,
                "author": "John Doe",
                "alt": "A beautiful sunset",
                "tags": "sunset",
                "image": "https://example.com/sunset.jpg",
                "description": "A beautiful sunset over the ocean."
            },
            {
                "id": 2,
                "author": "Jane Doe",
                "alt": "A majestic mountain",
                "tags": "mountain",
                "image": "https://example.com/mountain.jpg",
                "description": "A majestic mountain range in the distance."
            }
        ]

Retrieve a Photo by ID

Use this endpoint to retrieve a specific photo by its ID.

Endpoint URL

/photos/:id

HTTP Method

GET

URL Parameters

Name Description Required
id The ID of the photo to be retrieved Yes

Success Response

If the photo is found, the API will return the following:

HTTP Status Code Description
200 OK The photo was successfully retrieved

Error Response

If the photo is not found, the API will return the following:

HTTP Status Code Description
404 Not Found The photo could not be found

Posting photo API Documentation

Endpoint

POST /photos

Request Body

The API request must include the following parameters:

  • id (string, required)
  • author (string, required)
  • alt (string, required)
  • tags (string, optional)
  • image (string, required) - URL of the photo to be posted
  • description (string, optional) - description of the photo

Response

On success, the API will return a 201 status code.

On error, the API will return a console error with the error message.

Example

Below is an example of a successful photo post: 

        POST /photos
        {
        "id": "12345",
        "author": "John Doe",
        "alt": "A beautiful sunset",
        "tags": "sunset",
        "image": "https://example.com/sunset.jpg",
        "description": "A beautiful sunset over the ocean."
        }
        Response:
        201

Update a Photo

Use the PUT method to update the data of a photo identified by its id

Request URL

/photos/:id

Request Body

The request body should contain the following data in JSON format:

  • author: the author of the photo (string)
  • alt: the alternative text for the photo (string)
  • tags: tags associated with the photo, separated by comma (string)
  • image: the URL of the image (string)
  • description: a description of the photo (string)

Response

On success, the HTTP status code 200 (OK) will be returned.

On error, the HTTP status code 500 (Internal Server Error) will be returned.

Example

Here is an example of how to update a photo using cURL:

      curl -X PUT \
      -H "Content-Type: application/json" \
      -d '{ "id": "3", "author": "Jane Doe", "alt": "A beautiful sunset", "tags": "sunset, nature, orange", "image": "https://example.com/sunset.jpg", "description": "A breathtaking sunset over the ocean" }' \
      https://api.example.com/photos/123

Delete Photo Route

This route is used to delete a photo from the database.

URL

      DELETE /photos/:id

Request Body

No request body required for this route.

Response Body

Upon successful deletion, the response will be an empty JSON object with a status code of 200 OK. 

      {
        status: "success",
        data: {}
      }

Error Responses

If the photo with the given ID does not exist, the response will be a JSON object with a status code of 404 Not Found. 

      {
        status: "error",
        message: "Photo not found"
      }

If there is a server error, the response will be a JSON object with a status code of 500 Internal Server Error. 

      {
        status: "error",
        message: "Internal server error"
      }
